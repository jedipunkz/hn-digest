## Translation

タイトル: HN に質問: AI エージェントに仕様を与えて、無人で構築を開始させますか?
HN テキスト: 私は Cursor を使用しており、通常は計画モードにして仕様を入力します。これにより、計画をレビューして、実装が希望どおりに行われるかどうか、または計画が適切であるかどうかを確認できます。人々が仕様を入力し、エージェントに処理を任せ、その後、期待どおりに動作するかどうかを確認するために簡単なテストを行うという話を聞いたり見たりすると、それはあまりにも危険に思えます。エージェントが間違った仮定を立てて、私が必要としないものや要求していないものを含めたり、不必要に複雑なパスをたどるため実装中にエージェントを停止しなければならなかったという悪い経験が私には今でもあります。

## Original Extract

I use Cursor and I usually put it in plan mode and feed it the specs so I can review the plan and make sure the implementation will be done the way I want it to or to make sure the plan "feels" right. When I hear or see that people feed the specs, let agents handle things and then do some quick tests to make sure things work as expected, that sounds way too risky. I still have bad experiences where the agents made bad assumptions and included things I didn't need or ask for or had to stop the agent during implementation because it was going down a path that was needlessly complicated.
