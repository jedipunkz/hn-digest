HTML Deployer は、ChatGPT、Claude、Gemini から AI で生成された HTML を抽出し、それを迅速にダウンロードまたは公開できるようにします。
HTML Deployer は、AI が生成した HTML をライブ Web サイトに迅速かつ効率的に変換するツールです。
同じ拡張機能からプレビュー、ZIP エクスポート、直接パブリッシュを行うことで、1 つの合理化されたワークフローで HTML から Web サイトに移動できます。
HTML Deployer を使用して、Netlify、GitHub、FTP ホスティング、または独自のセルフホスト エージェントなどの Web サイト ターゲットに HTML ファイルをアップロードします。
## 主な機能 ⚙️
### AI が生成した HTML をチャット ページから抽出する
- ChatGPT、Claude、Gemini などのサポートされている AI プラットフォームで HTML コード ブロックを検出します。
- AI が生成した HTML を会話から直接抽出するため、最初にコードを別のエディターにコピーする必要はありません。
### 公開する前にプレビューする
- 生成されたページを公開する前にプレビューできるため、より自信を持って結果を確認できます。
- 公開する前に、デスクトップ、タブレット、モバイル スタイルの形式でページがどのように見えるかを確認してください。
- 展開前に、レイアウト、コンテンツ、および視覚的な構造を迅速にレビューします。
### ZIPをダウンロード
- ローカル ダウンロードまたは手動ホスティング アップロード用に、生成されたページを ZIP パッケージとしてエクスポートします。
- 独自のプロセスでアーカイブ、レビュー、引き継ぎ、またはアップロードできるように、ページのポータブル バージョンを保持します。
完全に使いたい場合は、ダウンロードしたファイルを使用してください

ページをどこでどのように公開するかを制御します。
### クラウドパブリッシング
1 つの拡張機能から Netlify や GitHub ベースのワークフローなどのクラウド ターゲットに公開します。
従来のホスティング パネルを管理せずに、ライブ URL へのより高速なパスが必要な場合は、このオプションを使用します。
### FTP展開
独自のホスティング アカウントまたはクライアント ホスティング環境を使用する場合は、FTP 経由で展開します。
既存のホスティング インフラストラクチャを直接制御したい場合は、このオプションを使用します。
### 自己ホスト型パブリッシング (ホスト エージェント)
独自のサーバーへの直接パスが必要な場合は、自己ホスト型ホスト エージェントを通じて公開します。
より柔軟で制御性の高いセルフホスト ワークフローが必要な場合は、このオプションを使用します。
### 繰り返しページを公開するために構築

## KEY FEATURES ⚙️
### Extract AI generated HTML from chat pages
- Detect HTML code blocks on supported AI platforms such as ChatGPT, Claude, and Gemini.
- Extract AI generated HTML directly from the conversation so you do not need to copy code into a separate editor first.
### Preview before you publish
- Preview the generated page before it goes live so you can review the result more confidently.
- Check how the page looks in desktop, tablet, and mobile style formats before publishing.
- Review layout, content, and visual structure faster before deployment.
### Download ZIP
- Export the generated page as a ZIP package for local download or manual hosting upload.
- Keep a portable version of the page for archiving, review, handoff, or uploading through your own process.
Use the downloaded file when you want full control over where and how the page is published.
### Cloud publishing
Publish to cloud targets such as Netlify and GitHub based workflows from one extension.
Use this option when you want a faster path to a live URL without managing a traditional hosting panel.
### FTP deployment
Deploy through FTP when you want to use your own hosting account or a client hosting environment.
Use this option when you prefer direct control over existing hosting infrastructure.
### Self hosted publishing (Host Agent)
Publish through a self hosted host agent when you want a direct path to your own server.
Use this option when you want a self host workflow with more flexibility and control.
### Built for repeated page publishing
- Keep deployment history so you can revisit recent pages and publishing activity.
- Auto suggest a filename
