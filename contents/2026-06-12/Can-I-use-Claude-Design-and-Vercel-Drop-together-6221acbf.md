クロードデザインとヴァーセルドロップを併用できますか？
URLをコピー クリップボードにコピーされました! 2026年6月11日 はい。 Claude Design はプロジェクトを .zip ファイルとしてエクスポートし、Vercel Drop はその .zip をブラウザにドラッグして展開します。エクスポートを解凍したり、Git に接続したり、ツールをインストールしたりする必要はありません。これらを組み合わせると、Claude のプロンプトが表示されます。

数分でライブ本番 URL に移行します。
見出しへのリンク Claude Design と Vercel Drop は何をしているのですか?
Anthropic Labs の Claude Design を使用すると、Claude との会話で希望を説明することで、Web サイト、プロトタイプ、プレゼンテーションを作成できます。 Claude Pro、Max、Team、Enterprise プランのリサーチ プレビューで利用でき、完成したデザインを Web サイト ファイルの .zip としてエクスポートします。
Vercel Drop は、ファイル、フォルダー、または .zip をブラウザーにドラッグすることでデプロイします。Git や Vercel CLI は必要ありません。新しい Vercel プロジェクトを作成し、ファイルをアップロードし、共有可能な URL を使用して本番環境に直接公開します。 Claude Design のエクスポートと Vercel Drop デプロイは、一方の出力がもう一方の入力とまったく同じであるため、互いに適合します。
見出しへのリンク Vercel Drop を使用して Claude Design プロジェクトをデプロイするにはどうすればよいですか?
デザインをエクスポートする : Claude Design で、右上隅の [エクスポート] を選択し、 [.zip としてダウンロード] を選択します。
ファイルをドロップします: vercel.com/drop に移動し、.zip をページにドラッグするか、ファイルのアップロードとして選択します。
名前を付けてデプロイ : デプロイ先の Vercel チームを選択し、プロジェクト名を入力して、 [デプロイ] を選択します。
プロンプトが表示されたら、ホームページを選択します。エクスポートのフォルダーの先頭にindex.htmlがない場合は、サイトのルートに読み込まれるページをルート (/) メニューから選択します。
デプロイメントが完了すると、サイトのライブ運用 URL を取得します。完全なチュートリアルは、「Vercel Drop を使用した Claude Design プロジェクトのデプロイ」にあります。
見出しへのリンク 最初に Claude Design のエクスポートを解凍する必要がありますか?
いいえ、Vercel Drop はファイルやフォルダーと一緒に .zip アーカイブを直接受け入れるため、Claude Design からダウンロードしたファイルをそのまま vercel.com/drop にドラッグできます。これは、ペアリングによって手順が節約される 1 つの場所です。一部の展開ツールでは、アーカイブを解凍する前に、フォルダー構造を確認する必要があります。

見出しへのリンク エクスポートにindex.htmlがない場合はどうすればよいですか?
Vercel Drop がこれを処理します。フォルダーの先頭にindex.htmlがない場合、Vercelはどのページを最初に見るべきかを尋ね、ルート（/）メニューからページを選択できるようにします。 [ルート ページなし] を選択すると、サイトのルート ( / ) は 404 を返しますが、Vercel は依然としてすべてのファイルを独自のパスで提供します。ファイルの名前を変更したり、エクスポートを再構築したりする必要はありません。
見出しへのリンク デザインを変更した後にサイトを更新するにはどうすればよいですか?
ドロップするたびに、新しい URL を持つ新しい Vercel プロジェクトが作成されるため、Claude Design プロジェクトを反復するには 2 つの方法があります。
ドロップを続ける : Claude Design から再エクスポートし、再度ドロップします。各バージョンは独自のプロジェクトと URL を取得します。これは、バージョンが独立しているプロトタイプに適しています。
リポジトリを接続する : Claude Design の Handoff to Claude Code エクスポートを使用して、プロジェクトをコードベースに移動し、GitHub、GitLab、または Bitbucket にプッシュし、リポジトリを Vercel に接続します。その後、すべてのプッシュが自動的にデプロイされ、プロジェクトの URL は変わりません。
探索中にドロップし続けてください。デザインが保守するサイトになったら、リポジトリに接続します。
見出しへのリンク 有料プランは必要ですか?
Vercel アカウントが必要です。Vercel Drop では無料のホビー プランが利用できます。クロード側では、クロード デザインはクロード プロ、マックス、チーム、エンタープライズ プランのリサーチ プレビュー段階にあるため、デザインを作成するにはそれらのいずれかが必要です。
見出しへのリンク 結論
Claude Design と Vercel Drop は、すぐに連携して使用できます。Claude Design から .zip をエクスポートし、 vercel.com/drop にドラッグすると、デザインが運用環境に反映されます。プロトタイプがプロジェクトに変わったら、反復を続け、それをコードベースに渡し、自動デプロイのために Git に接続します。

Can I use Claude Design and Vercel Drop together?
Copy URL Copied to clipboard! Jun 11, 2026 Yes. Claude Design exports your project as a .zip file, and Vercel Drop deploys that .zip by dragging it into your browser. You don't need to unzip the export, connect Git, or install any tooling. Together, they take you from a prompt in Claude to a live production URL in a couple of minutes.
Link to heading What do Claude Design and Vercel Drop do?
Claude Design, by Anthropic Labs, lets you create websites, prototypes, and presentations by describing what you want in a conversation with Claude. It's available in research preview on Claude Pro, Max, Team, and Enterprise plans, and exports finished designs as a .zip of website files.
Vercel Drop deploys a file, folder, or .zip by dragging it into your browser, with no Git or Vercel CLI required. It creates a new Vercel project, uploads your files, and publishes them straight to production with a shareable URL. A Claude Design export and a Vercel Drop deploy fit together because the output of one is exactly the input of the other.
Link to heading How do I deploy a Claude Design project with Vercel Drop?
Export your design : In Claude Design, select Export in the upper-right corner, then choose Download as .zip .
Drop the file : Go to vercel.com/drop and drag the .zip onto the page, or select it as a file upload.
Name and deploy : Choose the Vercel team to deploy to, enter a project name, and select Deploy .
Pick a homepage if prompted : If the export has no index.html at the top of the folder, choose which page loads at your site's root from the Root (/) menu.
When the deployment finishes, you get a live production URL for your site. The full walkthrough is in Deploy a Claude Design project with Vercel Drop .
Link to heading Do I need to unzip the Claude Design export first?
No. Vercel Drop accepts .zip archives directly, alongside files and folders, so you can drag the download from Claude Design onto vercel.com/drop as-is. This is one place the pairing saves a step: some deployment tools require you to unzip the archive and confirm the folder structure before uploading.
Link to heading What if my export doesn't have an index.html?
Vercel Drop handles this for you. If there's no index.html at the top of the folder, Vercel asks which page people should see first and lets you pick one from the Root (/) menu. If you choose No root page , your site's root ( / ) returns a 404, but Vercel still serves every file at its own path. You don't need to rename files or restructure the export.
Link to heading How do I update my site after changing the design?
Each drop creates a new Vercel project with a new URL, so you have two ways to iterate on a Claude Design project:
Keep dropping : Re-export from Claude Design and drop again. Each version gets its own project and URL, which works well for prototypes where versions stand alone.
Connect a repository : Use Claude Design's Handoff to Claude Code export to move the project into a codebase, push it to GitHub, GitLab, or Bitbucket, and connect the repository to Vercel . After that, every push deploys automatically and your project URL stays the same.
Keep dropping while you're exploring; connect a repository once the design becomes a site you maintain.
Link to heading Do I need a paid plan?
You need a Vercel account, and the free Hobby plan works for Vercel Drop. On the Claude side, Claude Design is in research preview on Claude Pro, Max, Team, and Enterprise plans, so creating the design requires one of those.
Link to heading The bottom line
Claude Design and Vercel Drop work together out of the box: export a .zip from Claude Design, drag it onto vercel.com/drop , and your design is live in production. When a prototype turns into a project you'll keep iterating on, hand it off to a codebase and connect Git for automatic deployments.
