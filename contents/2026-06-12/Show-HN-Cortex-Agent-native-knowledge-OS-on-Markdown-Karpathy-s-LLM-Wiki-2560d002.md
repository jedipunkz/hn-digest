Markdown 上に構築されたエージェントネイティブのナレッジ OS
PULSE8.ai Cortex は、Markdown 上に構築されたエージェントネイティブのナレッジ OS です。これは、AI エージェントと人間に、型付きナレッジ グラフ、全文検索、MarkItDown を利用したコンパイラを基盤とした共有ボールトを提供します。これらはすべて、統合された MCP インターフェイスを通じてアクセスできます。
ファイル (PDF、DOCX、PPTX、XLSX、HTML、画像など) をドロップすると、エージェントは知識の読み取り、書き込み、検索、リンク、コンパイルを行うことができます。データベースは必要ありません。
Andrej Karpathy の LLM Wiki パターンに触発されており、クエリごとに再導出されるのではなく、LLM によって維持される永続的で複合的な知識ベースです。トビ・ルーによる検索

tke の QMD 。
PULSE8.ai Cortex には Docker が必要です。 OpenRouter API キーはオプションであり、LLM を利用したウィキ記事間の相互参照にのみ必要です。ファイル変換は、API キーなしですぐに機能します。
git clone https://github.com/synpulse8-opensource/pulse8-ai-cortex-knowledge-vault.git
cd 皮質-知識-保管庫
PULSE8.ai Cortex を起動します。
./scripts/start.sh
これにより、**PULSE8.ai Cortex** (`:8420` の API + MCP) と **QMD** (`:3100` で検索) の両方がビルドおよび開始され、ヘルス チェックを待って準備完了です。
MCP クライアント (Claude Desktop など) を http://localhost:8420/mcp/ に接続します。
Cortex-only モード (macOS / ネイティブ QMD)
QMD をネイティブに実行したい場合 (例: Metal GPU アクセラレーションを備えた macOS 上)、Cortex コンテナーのみを起動します。
# ターミナル 1: QMD をネイティブに実行する
npm install -g @tobilu/qmd
VAULT_PATH=./example_vault ノード docker/qmd/server.mjs
# ターミナル 2: Docker で Cortex のみを起動する
./scripts/start.sh --cortex-only
停止するには: ./scripts/stop.sh --cortex-only
GPU アクセラレーション QMD (EC2 / Linux と NVIDIA GPU)
NVIDIA GPU アクセラレーションを使用した運用環境の場合:
docker compose -f docker-compose.yml -f docker-compose.gpu.yml up --build -d
インスタンスの選択、NVIDIA ツールキットのインストール、コストの見積もりに関する完全なガイドについては、docs/ec2-gpu-setup.md を参照してください。
ナレッジグラフ
型付きグラフ エンジン (NetworkX) — ウィキリンク、タグ、およびカスタム エッジ。ファイルが変更されるたびに自動的に維持されます。
全文検索
デフォルトでハイブリッド (BM25 + ベクトル + 再ランキング) を使用した QMD 検索。キーワードおよびセマンティック モードを選択可能。結果は構成可能な TTL でキャッシュされます。

生のソース (PDF、DOCX、PPTX、XLSX、HTML、画像など) を MarkItDown 経由で Markdown に変換します。 LLM は相互参照のみに使用されます。

ボールトのレイアウト
ボールトは、目的別に編成された Markdown ファイルのプレーンなディレクトリです。 Cortex は、各ファイルを、グラフ エンジンによって使用され、REST および MCP 応答で公開される型付きノード ( NodeType ) に分類します。
優先順位 (最初の一致が勝ち):
* Frontmatter type:* — 明示的なオーバーライドが常に優先されます (例: Agents/foo.md の type: note は NodeType.NOTE に解決されます)
Folder prefix — files under raw/ agents/ sessions/ daily/ feedback/ inherit the folder's type with no filename suffix needed (e.g. daily/2026-06-10.md → daily )
ファイル名のサフィックス (下位互換性) — .agent.md 、 .session.md 、 .memory.md はどこでも引き続き受け入れられます (例: wiki/legacy.agent.md → Agent_def )。
実際には、これは YYYY-MM-DD.md を daily/ に直接ドロップするか、接尾辞のない planner.md を Agents/ にドロップすることができ、グラフと API は名前を変更せずにそれらを正しく分類することを意味します。
すべての vault_write 、 vault_ingest 、および成功したコンパイル イベント (MCP および REST パス) は、daily/YYYY-MM-DD.md にある今日の UTC 日次メモに自動的にミラーリングされます。ファイルはその日の最初のイベントで作成され、その後のイベントごとに ## [HH:MM] イベント | が追加されます。概要ブロックと [[wiki-stem]] ウィキリンク (つまり、ウォッチャーは影響を受けるノートに LINKS_TO エッジを描画します)。形式は、Karpathy log.md の grepable-prefix パターンに従います。grep "^## \[" daily/2026-06-10.md は、その日のきれいなタイムラインを示します。
daily/ 、 Facebook/ 、または .cortex/ をターゲットとする書き込みは、意図的にミラーリングされません (自己参照ノイズとなる可能性があります)。非表示の .cortex/log.md 監査ログは影響を受けず、引き続きすべての操作を受信します。
一度に多くのファイル (数十または数百の PDF、論文、ドキュメント) を取り込む場合は、次を使用します。

MCP を介して一度に 1 つずつシェル スクリプトを入力する代わりに、ワンクリックでシェル スクリプトを実行します。ローカル ディレクトリから直接読み取ります。ワイヤ オーバーヘッドや実行中のサーバーは必要ありません。SHA-256 ハッシュを介して重複を排除し、制限付き同時実行でコンパイルし、最後にインデックスを 1 回再構築します。
ワンクリック スクリプト (推奨)
# ディレクトリからすべてのファイルを取り込む
./scripts/bulk_ingest.sh ./my-papers/
# 取り込まれる内容をプレビューするためのドライラン
./scripts/bulk_ingest.sh ./my-papers/ --dry-run
# 強制的に再取り込み (重複排除マニフェストをバイパス)
./scripts/bulk_ingest.sh ./my-papers/ --force
# LLM 同時実行性の制御 (デフォルト: 4)
./scripts/bulk_ingest.sh ./my-papers/ --concurrency 8
このスクリプトは、LLM キーとボールト パスの .env を自動的にロードし、概要を出力してから、完全なパイプライン (コピー、コンパイル、再インデックス) を実行します。 Cortex サーバーを実行する必要はありません。
CORTEX_VAULT_PATH=./example_vault uv run Cortex-bulk-ingest --source ./my-papers/
Docker の内部
# .env に INGEST_DIR を設定するか、エクスポートしてから再起動します
INGEST_DIR=/path/to/your/papers をエクスポート
ドッカー構成 -d
# コンテナ内で一括取り込みを実行する
docker execpulse8-ai-cortex uv run Cortex-bulk-ingest --source /ingest
REST API経由
MCP を使用せずにプログラムで使用する場合 (Cortex サーバーを実行する必要があります):
カール -X POST http://localhost:8420/api/v1/bulk-ingest \
-H " Content-Type: application/json " \
-H " x-API キー: 秘密の API キー " \
-d ' {"source_dir": "/ingest", "concurrency": 4} '
重複排除
重複排除マニフェストは .cortex/ingest-manifest.json に保存されます。ファイルはファイル名ではなくコンテンツのハッシュによって照合されます。ファイルの名前を変更しても再取り込みは行われず、別の名前の同じコンテンツはスキップされます。
例をコピーし、値を入力します。
cp .env.example .env

PULSE8.ai Cortex is an agent-native knowledge OS built on Markdown. It gives AI agents and humans a shared vault backed by a typed knowledge graph, full-text search, and a MarkItDown -powered compiler — all accessible through a unified MCP interface.
Drop files in (PDF, DOCX, PPTX, XLSX, HTML, images, and more), let agents read, write, search, link, and compile knowledge — no database required.
Inspired by Andrej Karpathy 's LLM Wiki pattern — a persistent, compounding knowledge base maintained by LLMs instead of re-derived on every query. Search powered by Tobi Lütke 's QMD .
PULSE8.ai Cortex requires Docker. An OpenRouter API key is optional — needed only for LLM-powered cross-referencing between wiki articles. File conversion works out of the box without any API key.
git clone https://github.com/synpulse8-opensource/pulse8-ai-cortex-knowledge-vault.git
cd cortex-knowledge-vault
Launch PULSE8.ai Cortex:
./scripts/start.sh
This builds and starts both **PULSE8.ai Cortex** (API + MCP on `:8420`) and **QMD** (search on `:3100`), waits for health checks, and you're ready to go.
Connect your MCP client (e.g. Claude Desktop) to http://localhost:8420/mcp/ .
Cortex-only mode (macOS / native QMD)
If you want QMD to run natively (e.g. on macOS with Metal GPU acceleration), start only the Cortex container:
# Terminal 1: Run QMD natively
npm install -g @tobilu/qmd
VAULT_PATH=./example_vault node docker/qmd/server.mjs
# Terminal 2: Start only Cortex in Docker
./scripts/start.sh --cortex-only
To stop: ./scripts/stop.sh --cortex-only
GPU-accelerated QMD (EC2 / Linux with NVIDIA GPU)
For production deployments with NVIDIA GPU acceleration:
docker compose -f docker-compose.yml -f docker-compose.gpu.yml up --build -d
See docs/ec2-gpu-setup.md for a full guide on instance selection, NVIDIA toolkit installation, and cost estimates.
Knowledge Graph
Typed graph engine (NetworkX) — wikilinks, tags, and custom edges, auto-maintained on every file change
Full-Text Search
QMD search with hybrid (BM25 + vector + re-ranking) by default; keyword and semantic modes selectable. Results cached with a configurable TTL.
File Compiler
Converts raw sources (PDF, DOCX, PPTX, XLSX, HTML, images, etc.) to Markdown via MarkItDown . LLM used only for cross-referencing.
MCP Server
Streamable HTTP + stdio transport — works with Claude Desktop, Cursor, and any MCP client
Feedback & Notifications
vault_feedback captures quality feedback as notes; optional Microsoft Teams webhook posts an adaptive card per submission
Daily Activity Log
Every write/ingest/compile is mirrored into daily/<date>.md as a greppable, wikilinked timeline
Bulk Ingest
Ingest dozens or hundreds of files at once from a local directory with SHA-256 dedup and bounded concurrency
REST API
FastAPI endpoints mirroring all MCP tools at /api/v1/ , including multipart file upload and bulk ingest
Vault Watcher
Real-time filesystem monitoring — graph stays in sync automatically
Zero Database
Everything persists as Markdown + JSON on your filesystem

Vault layout
The vault is a plain directory of Markdown files organised by purpose. Cortex classifies each file into a typed node ( NodeType ) used by the graph engine and exposed in REST and MCP responses.
Order of precedence (first match wins):
* Frontmatter type:* — explicit override always wins (e.g. type: note in agents/foo.md resolves to NodeType.NOTE )
Folder prefix — files under raw/ agents/ sessions/ daily/ feedback/ inherit the folder's type with no filename suffix needed (e.g. daily/2026-06-10.md → daily )
Filename suffix (backward-compatible) — .agent.md , .session.md , .memory.md are still honored anywhere (e.g. wiki/legacy.agent.md → agent_def )
In practice this means you can drop YYYY-MM-DD.md straight into daily/ , or an unsuffixed planner.md into agents/ , and the graph and API will classify them correctly without any renaming.
Every vault_write , vault_ingest , and successful compile event (MCP and REST paths) is automatically mirrored into today's UTC daily note at daily/YYYY-MM-DD.md . The file is created on first event of the day and each subsequent event appends a ## [HH:MM] event | summary block plus a [[wiki-stem]] wikilink (so the watcher draws a LINKS_TO edge to the affected note). The format follows the Karpathy log.md greppable-prefix pattern — grep "^## \[" daily/2026-06-10.md gives a clean timeline of the day.
Writes targeting daily/ , feedback/ , or .cortex/ are deliberately not mirrored (would be self-referential noise). The hidden .cortex/log.md audit log is unaffected and continues to receive every operation.
For ingesting many files at once (dozens or hundreds of PDFs, papers, docs), use the one-click shell script instead of feeding them one at a time through MCP. It reads directly from a local directory — no wire overhead, no running server required — deduplicates via SHA-256 hashing, compiles with bounded concurrency, and rebuilds the index once at the end.
One-click script (recommended)
# Ingest all files from a directory
./scripts/bulk_ingest.sh ./my-papers/
# Dry-run to preview what would be ingested
./scripts/bulk_ingest.sh ./my-papers/ --dry-run
# Force re-ingest (bypass dedup manifest)
./scripts/bulk_ingest.sh ./my-papers/ --force
# Control LLM concurrency (default: 4)
./scripts/bulk_ingest.sh ./my-papers/ --concurrency 8
The script automatically loads your .env for the LLM key and vault path, prints a summary, then runs the full pipeline (copy, compile, reindex). No running Cortex server needed.
CORTEX_VAULT_PATH=./example_vault uv run cortex-bulk-ingest --source ./my-papers/
Inside Docker
# Set INGEST_DIR in .env or export it, then restart
export INGEST_DIR=/path/to/your/papers
docker compose up -d
# Run bulk ingest inside the container
docker exec pulse8-ai-cortex uv run cortex-bulk-ingest --source /ingest
Via REST API
For programmatic use without MCP (requires running Cortex server):
curl -X POST http://localhost:8420/api/v1/bulk-ingest \
-H " Content-Type: application/json " \
-H " x-api-key: your-secret-api-key " \
-d ' {"source_dir": "/ingest", "concurrency": 4} '
Deduplication
The dedup manifest is stored at .cortex/ingest-manifest.json . Files are matched by content hash, not filename — renaming a file won't cause re-ingestion, and the same content under a different name will be skipped.
