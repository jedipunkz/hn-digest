Claude Design は、プロジェクト全体でデザイン システムに固執し、Claude Code と流動的に連携し、キャンバス上で直接編集できるようになり、すでに使用しているより多くのツールに接続できるようになりました。 Claude デスクトップ アプリのサイドバーに新しいホームが追加されました。 claude.ai/design でも見つけることができます。
私たちの目標は以前と同じです。それは、通常は期限内に許可されている以上に多くの方向性を検討できるようにし、磨き上げや構築への引き継ぎを可能な限りシームレスに行うことです。この目標は現在、より多くの人々に貢献しています。最初の 1 週間で 100 万人以上が Claude Design を使用し、あなたのフィードバックが私たちが次に構築するものを形作っています。
設計システムのコンポーネントを使用して構築する
柔軟性を高め、精度を向上させるために、設計システムのインポートを再構築しました。 GitHub リポジトリ、デザイン ファイル、または生のアップロードから 1 つまたは複数のデザイン システムを取り込みます。 Claude はコンポーネントを使用してビルドし、その出力を設計システムと照合してチェックし、ユーザーが確認する前に修正を加えます。大規模なチームの場合、新しい管理者ロールは 1 つの標準を承認できます。

システムとロックダウン編集により、作業は常に会社のガイドラインに一致します。
本日公開されるアップデートにより、作業の同期を維持しながら、Claude Design と Claude Code の間での作業を簡単に移行できるようになりました。 /design-sync を使用してデザイン システムを取り込むと、Claude Design で構築するものはすべて既存のコンポーネントから開始されます。デザインをソフトウェアにする準備ができたら、それを Claude Code に引き渡すことができます。これにより、スクリーンショットからやり直すのではなく、既存の作業が継続されます。
クロードコードから始めたいですか? /design を使用すると、端末から離れることなくデザイン プロジェクトを作成、編集、同期できます。設計をコードベースにインポートしたり、コードをライブ プロトタイプに変換したり、クロードにプロジェクトを最後まで実行させたりできます。
「Claude Design は、実証済みの真のデザイン ツールの機能と機能を備えたフロンティア モデルのインテリジェンスを私に与えてくれます。Tenex のサイト、新しいブランド アセット、またはプレゼンテーションのデザイン ディレクションに取り組むときはいつでも、Claude Design が最初に行きます。親しみやすい UX と強いセンスとデザインの本能の組み合わせが、Claude Design が私の技術スタックの中核部分になっている理由です。そして、Claude Design と Claude Code の間の引き継ぎにより、プロトタイプから製品化までのプロセスがシームレスになります。」
- Alex Lieberman 氏、Morning Brew & Tenex 共同創設者
新しいエディターを使用すると、デザインのあらゆる要素を直接、きめ細かく制御できます。新しい豊富なレイアウト コントロールにより、要素をドラッグ、サイズ変更、位置合わせできます。何百もの安定性に関する修正により、エディターは実際の使用にも耐えられるようになりました。
また、Claude Design はチャット、Claude Cowork、および Claude Code と使用制限を共有するようになりました。そのため、ほとんどの人が作成する余地が大幅に広がり、制限に達する頻度が減ります。平均ターンでは、同じ結果を達成するために使用するトークンの量が減り、エラーが減少します。

「Replit と Anthropic は、誰でも、どこでも、AI を使って自分のアイデアを実現できるべきだという信念を共有しています。私たちの共通の目標は、アイデアがどこから始まってもビルダーと出会うことです。Claude Design MCP の統合は自然な次のステップです。ビルダーは、Claude Design でブランド上のアプリをデザインし、Replit で構築、改良、出荷することが、すべて 1 つのシームレスなエクスペリエンスで可能になります。」
「Lovable では、問題に最も近い人々が自ら問題を解決する力を持っているべきだと考えています。Claude Design とのパートナーシップを通じて、さらに多くの人々が自分の周りの世界のための解決策を構想しているのを見ることに興奮しています。現在、彼らはそれらのアイデアを Lovable 内で実稼働対応のアプリケーションとしてシームレスに実現できるようになりました。」
「今ではデッキを生成するのは簡単ですが、それを自分のものにするのは常に難しい部分です。Claude Design を Gamma に接続すると、そのギャップが縮まります。私たちはこれを Anthropic で構築しました。そのため、Claude でデザインしたものはすべて、完全に編集可能な状態で、さらに進めたいと思った瞬間に Gamma に直接移行できます。」
「Wix は常にデザイナーにとってホームでした。私たちは Anthropic でそれをさらに一歩進めることに興奮しています。Claude Design は Wix Headless にシームレスに接続し、デザイナーがクリエイティブ プロセスを離れることなく Wix のバックエンド インフラストラクチャに直接アクセスできるようにします。デザイン コミュニティは、UX ビジョンを最初のスケッチから完全に機能するスケーラブルな製品まで、まさに想像どおりに実現できるようになりました。」

Claude Design now sticks to your design system across projects, works fluidly with Claude Code, lets you edit directly on the canvas, and connects to more tools you already use. It has a new home in the sidebar on the Claude desktop app; you can also find it at claude.ai/design .
Our goal remains the same as before: letting you explore more directions than a deadline usually allows and making the handoff to polish or build as seamless as possible. That goal now serves a lot more people. Over one million people used Claude Design in its first week, and your feedback is shaping what we build next.
Build with your design system components
We rebuilt the design system import to give you more flexibility and increase precision. Bring in one or several design systems from a GitHub repo, design files, or raw uploads. Claude builds with your components, checks its output against your design system, and makes corrections before you see it. For larger teams, a new admin role can approve one standard system and lock down edits, so the work always matches your company guidelines.
It's now easier to move between working in Claude Design and Claude Code while keeping your work synced, with updates rolling out today. Use /design-sync to pull in your design system, so everything you build in Claude Design starts from your existing components. When a design is ready to become software, you can hand it off to Claude Code, which continues from your existing work instead of starting over from a screenshot.
Prefer to begin in Claude Code? You can create, edit, and sync design projects without leaving the terminal using /design. Import a design into your codebase, turn your code into a live prototype, or let Claude carry a project all the way through.

Claude Design also now shares usage limits with chat, Claude Cowork, and Claude Code, so most people will get much more headroom to create, and those hitting limits will do so less often. The average turn now uses fewer tokens to achieve the same results, and errors are down sharply.
