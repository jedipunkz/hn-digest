AdBuster 2.0 PRO – TV 音量スタビライザー
ML/AI と CEPA ロジックを搭載したオフライン オーディオ安定化エンジン。
⚠️ CEPA Logic は、このリポジトリに含まれる追加の独自ライセンスの対象となります。
参照: CEPA_LOGIC_LICENSE.txt
うるさいテレビ広告を自動的に下げて復元する小さな Windows ツール

通常のボリュームをスムーズに。
AdBuster PRO は、テレビの音声をリアルタイムで監視し、大音量のコマーシャルが表示されると自動的に音量を下げる軽量の Windows アプリケーションです。
通常の音声に戻ると、スムーズかつ自然な音量に戻ります。
雲はありません。
アカウントがありません。
テレメトリはありません。
データ収集はありません。
すべてが PC またはラップトップ上でローカルに実行されます。
「リリース」ページから最新のリリースをダウンロードします。
PC 上の任意の場所にフォルダーを解凍します。
Start.bat を実行すると、すべてのモジュールが自動的に起動します。
(オプション) Broadlink IR デバイスを音量制御用に設定します。
テレビの視聴を開始します。AdBuster は大音量の広告を検出し、リアルタイムで音声を安定させます。
🔊 リアルタイムのラウドネス検出
📉 コマーシャル中の自動音量低減
🔄 広告後のスムーズな音量復元
📡 IR ブラスターのサポート (Broadlink RM シリーズ)
⚙️ 設定可能なしきい値と反応時間
🖥️ バックグラウンドで静かに実行されます
🔐 完全にローカル — インターネットは不要
AdBuster 2.0 PRO は一般的なオーディオ ツールではありません。
これは、以下を組み合わせた唯一の Windows アプリケーションです。
🔹 リアルタイムオーディオ分析 (DSP)
専用のオーディオ エンジンは、完全にオフラインで、ラウドネス、安定性、スパイク、長期パターンをリアルタイムで抽出します。
🔹 CEPA ロジック — 人間のような意思決定フレームワーク
人間と同じ方法で音声イベントを解釈する独自の文脈推論レイヤー:
状況や時間帯に適応する
🔹 オフライン機械学習 (無料 + プロ)
model.pkl — 軽量の短期分類子
model_deep.pkl — PRO レベルの長期パターン認識ツール
AdBuster は、次の機能を備えた唯一の Windows ツールです。
IR コマンドを自動的に送信します
ユーザーの操作なしでテレビの音量を安定させます
すべての処理はユーザーのデバイス上で行われます。
🔹 独自の CEPA + ML + IR パイプライン
この組み合わせは他の Windows には存在しません

アプリケーションまたはスマートホーム ツール。
AdBuster 2.0 PRO は、完全でインテリジェントなプライバシー最優先のオーディオ安定化システムとして独立しています。
マイクはテレビの音声を聞きます。
突然の音量の急上昇が検出されると (広告によく見られます)、AdBuster PRO は音量を下げるかどうかを決定します。
アプリはローカルの Flask サーバー (VolMaster) にコマンドを送信します。
VolMaster は IR コマンドを Broadlink デバイスに転送します。
Broadlink は IR 信号をテレビに送信します。
音声が通常に戻ると、アプリは以前の音量レベルを復元します。
すべてのロジックはローカルで処理され、クラウドや外部サーバーは使用されません。

📞 IRコマンドポリシー
AdBuster の自動化レイヤーは 2 つの IR コンポーネントのみを使用します

コマンド:
これらは、CEPA、ML コントローラー、アプリ ロジックによって生成される唯一のコマンドです。
MUTE および POWER コマンドは手動のみです (UI ボタン)
自動化パイプラインでは決して使用されません。
自動音量変更は常に次の制約を受けます。
🧩 アーキテクチャの概要 — 3 つのモジュール + CEPA Brain
AdBuster 2.0 PRO は、リアルタイムで連携する 3 つの独立したモジュールから構築されています。
その中心には、すべてを結び付ける意思決定の頭脳である CEPA が座っています。
マイクから音声をキャプチャします
RMS、ピーク、ダイナミクス、ドリフト、スパイクを計算します
短期および長期のラウドネス履歴を構築
CEPA 向けに構造化されたオーディオ イベントを生成します。
スムージング、デッドゾーン、安定ゾーン、クールダウン、安全制限を処理します
役割: リアルタイム検出 + ランタイム ロジック
🧿 Aduster ML — 機械学習エンジン
オンデバイスモデル(model.pklおよびmodel_deep.pkl)を使用します。
RMS履歴に基づいてAD / NORMAL動作を分類します
CEPAの解釈を検証する
IRコマンドを送信しません
音量を制御しません
役割: パターンの検証 + 長期的な統計的コンテキスト
🧰 VolMaster — IR コントロールサーバー
Broadlink RM デバイスと通信するローカル Flask サーバー。
VOL_UP / VOL_DOWN IR コマンドを送信します。
あらゆる IR ベースの TV またはオーディオ デバイスと互換性があります。
/send?cmd=... および /status エンドポイントを公開します。
役割: IR コマンドの配信を担当する実行層。
🧠 CEPA — 意思決定脳
CEPA (コンテキスト イベント パターン分析) は別個のモジュールではありません。
これは、AdBuster 2.0 PRO 内の内部インテリジェンス レイヤーです。
CEPA は以下を受け取ります:
AdBuster からのオーディオ イベント
アダスターMLからのAD/NORMAL状態
アプリロジックからの最近のアクション、クールダウン、安全制限
CEPA は以下を実行します。
文脈パターン分析
人間のようなフィルタリング
異常検出
安定性評価
安全執行
CEPAが決定

簡単に言うと:
AdBuster が検出 → Aduster ML が検証 → CEPA が決定 → VolMaster が実行。

A microphone listens to the TV audio.
When a sudden loudness spike is detected (typical for ads), AdBuster PRO decides whether to lower the volume.
The app sends a command to the local Flask server (VolMaster).

Broadlink sends the IR signal to the TV.
When audio returns to normal, the app restores the previous volume level.

🧩 Architecture Overview — 3 Modules + CEPA Brain
AdBuster 2.0 PRO is built from three independent modules that work together in real time.
At the center sits CEPA, the decision‑making brain that connects everything.
Captures audio from the microphone
Computes RMS, peaks, dynamics, drift and spikes
Builds short‑ and long‑term loudness history
Generates structured audio events for CEPA
Handles smoothing, deadzone, stable zone, cooldowns and safety limits
Role: real‑time detection + runtime logic
🧿 Aduster ML — Machine Learning Engine
Uses on‑device models (model.pkl and model_deep.pkl)
Classifies AD / NORMAL behaviour based on RMS history
Validates CEPA’s interpretation
Does not send IR commands
Does not control volume
Role: pattern validation + long‑term statistical context
🧰 VolMaster — IR Control Server
Local Flask server that communicates with Broadlink RM devices.
Sends VOL_UP / VOL_DOWN IR commands.
Compatible with any IR‑based TV or audio device.
Exposes /send?cmd=... and /status endpoints.
Role: execution layer responsible for IR command delivery.
🧠 CEPA — The Decision‑Making Brain
CEPA (Contextual Event Pattern Analysis) is not a separate module.
It is the internal intelligence layer inside AdBuster 2.0 PRO.
CEPA receives:
audio events from AdBuster
AD / NORMAL state from Aduster ML
recent actions, cooldowns and safety limits from the app logic
CEPA performs:
contextual pattern analysis
human‑like filtering
anomaly detection
stability evaluation
safety enforcement
CEPA decides:
VOL_DOWN
VOL_UP
PASS
BLOCK
AdBuster then forwards the final command to VolMaster, which executes it via IR.
In simple terms:
AdBuster detects → Aduster ML validates → CEPA decides → VolMaster executes.
