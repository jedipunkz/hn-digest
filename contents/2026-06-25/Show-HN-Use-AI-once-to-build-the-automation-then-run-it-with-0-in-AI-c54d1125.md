If you want to work more deeply with your existing AI tools and desktop workflows, we offer optional extensions for flexible access. The Automation Skill Builder application must still be running locally (often in the background); extensions only embed the web UI—point them at the same URL you use in a browser.

search for Automation Skill Builder and set
automationSkillBuilder.baseUrl to your running service (for example
http://127.0.0.1:8800/ ).
From a VSIX: Open Extensions → ⋯ →
Install from VSIX… , choose the file, then set
automationSkillBuilder.baseUrl the same way.
The same side-panel package works in Google Chrome and Microsoft Edge (Chromium). Download the ZIP, extract it, then load the folder as an unpacked extension.
Unzip so the folder contains manifest.json . Open chrome://extensions or edge://extensions , turn on Developer mode , click Load unpacked , and select that folder. In the side panel, set host/port to match your running app (defaults align with 127.0.0.1:8800 ).
Reminder: Extensions do not start the backend. Keep Automation Skill Builder running on your machine and use the same address and port in the extension as in your browser so the embedded UI can load.
