彼女はあなたのことを覚えています。階層化されたメモリ システムは、ユーザーのスタイル、慣習、決定、および以前に修正したバグを学習します。専用のメモリ エージェントによってキュレーションされるため、コンテキストが無駄なく、リコールが鮮明に保たれます。ローカルファーストのオプトインセマンティックリコール。
彼女は建築する前に考えます。 7 つのモードにわたる 24 人のスペシャリスト ペルソナが、実行前に計画、検証、挑戦します。単純な質問ではオーケストレーションが完全にスキップされ、オーバーヘッドがゼロになります。
彼女は作業に適した艦隊を選択します。仕事を選ぶのはあなたです。適切なマルチモデル フリートはそこから外れます (「モデルとフリート」を参照)。
彼女は無料で教えてくれます。教えるモードに切り替えて、「Rust を教えて」と言えば、彼女はパーソナライズされたカリキュラムを構築し、ライブ Web と照らし合わせて事実確認しながら、レッスンごとにそれを教えます。他のすべてのモードと同じクレジット。

ペルソナ — Ava の社内スペシャリスト チーム
タスクが複雑な場合、Ava の社内チームが活動します。各ペルソナは、独自のクリーンなコンテキストと範囲指定されたツールを取得します (チャレンジャーは読み取りますが、書き込みは行いません)。単純なタスクは直接実行されます。
コード: スカウト → アーキテクト → 検証者 → シーケンサー → チャレンジャー → ビルダー
プラン：研究者→建築家→挑戦者

ブレーンストーミング: 探索者 → 研究者 → アイデア者 → 挑戦者 → 洗練者
Chat と Write はペルソナなしで実行されます。Ava だけです。
モデルを選ぶのではなく、仕事を選ぶのです。そうすれば、その中から適切なフリートが選ばれます。本日は 2 つの組織化された艦隊が出荷されます。 3番目のものが展開されています。

任意のプロバイダーに独自のキーを持ち込むか、プランで Ava 管理のモデルを使用します。すべてのプランにはすべての管理対象モデルが含まれます。階層は月々のクレジット許容量のみが異なり、使用できるモデルは異なります。

最初の起動時に、Ava は対話型のセットアップ ウィザードを実行します。プロバイダーを選択し、キーを貼り付けます (またはスキップして無料枠にサインインします)。これで準備は完了です。
インストール

VS Code Marketplace から入手するか、[拡張機能] パネルで「Ava Supernova」を検索します。アクティビティバーの Ava アイコンをクリックして (または Ctrl+Shift+A )、開始します。
ava-supernova.com からスタンドアロン デスクトップ IDE をダウンロードします。ローカル Node.js エンジンは、マシン上で @ava/core を実行します。これは、ブラウザーや VS Code を必要としない完全なエージェントです。

ツール
60 以上のユーザー向け組み込みツール。 Ava はどれを使用するかを決定し、実行し、結果を読み取り、推論を続けます。 (IDE は、デスクトップ モードとブラウザ自動化ツールを上部に追加し、デスクトップ モードにゲートします。)
Ava は実際のアセットを生成し、それらをプロジェクトに直接接続します。ロゴは public/ に、ヒーロー クリップはランディング ページに組み込まれます。
画像とビデオは Wan (Alibaba のオープンウェイト モデル) で実行され、クリーンなグラフィック デザイン出力と同期されたオーディオ付きの短いクリップ用に構築されています。
すべてのアセットはライブラリ内で追跡されます。グリッドまたはリスト ビュー、タイプによるフィルター、プレビュー、プロジェクト間での再利用が可能です。
OS キーチェーン (VS Code SecretStorage) で暗号化された API キー。ログに記録されることも、Ava サーバーに送信されることもありません。
会話と記憶はローカルに保存されます ( ~/.ava/ )。デフォルトではローカルファースト。
認証情報の安全性 - ランタイム検出により、API キー、トークン、秘密キーがメモリに保存されるのをブロックします。信頼性の高いキー パターンは、画面に表示される前にチャット出力から編集されます。
機能ベースのシークレット — Ava は、現在のセッションをスコープとする生の値ではなく、ボールト シークレットの不透明なハンドルを受け取ります。
即時注入耐性が組み込まれています。テレメトリーなし - 使用状況、コード、会話は追跡されません。
クラウド同期はオプトインであり、差分追跡機能があるため、何がプッシュされようとしているかを正確に確認できます。
すべてのプランはすべての管理対象モデルを取得します。レベルの違いは月々のクレジット枠のみです。
あるいはフル稼働

主要な設計上の決定事項:
共有コア、ゼロ重複 - CLI、拡張機能、IDE はすべて同じ @ava/core エンジン、ツール、プロバイダーを実行します。
イベント駆動型エージェント — エージェントは、すべてのホストがサブスクライブする型指定されたイベントを発行します。
自己完結型ツール - 各ツールは独自のスキーマとハンドラーを持ちます。 1 つ追加すると、1 つのファイルと 1 つのレジストリ行が追加されます。
プロバイダー SDK はありません。すべてのプロバイダー通信はネイティブ fetch を使用します。
権限マトリックス - リスク レベル (安全、書き込み、危険) を権限モードと組み合わせて、何が承認が必要かを決定します。計画とユーザーの質問には常に承認が必要です。
ローカルファースト — すべてがオフラインで動作します。クラウド同期はオプトインです。
貢献は歓迎です — これはコミュニティのために構築されています。

Tools
60+ user-facing built-in tools. Ava decides which to use, runs them, reads the results, and keeps reasoning. (The IDE adds desktop- and browser-automation tools on top, gated to Desktop mode.)
Ava generates real assets and wires them straight into your project — a logo into public/ , a hero clip into your landing page.
Images & video run on Wan (Alibaba's open-weight models), built for clean graphic-design output and short clips with synchronised audio.
Every asset is tracked in the Library — grid or list view, filter by type, preview, reuse across projects.
API keys encrypted in your OS keychain (VS Code SecretStorage). Never logged, never sent to Ava servers.
Conversations & memory stored locally ( ~/.ava/ ). Local-first by default.
Credential safety — runtime detection blocks API keys, tokens, and private keys from being saved to memory; high-confidence key patterns are redacted from chat output before they hit the screen.
Capability-based secrets — Ava receives opaque handles for vault secrets, not raw values, scoped to the current session.
Prompt-injection resistance built in. No telemetry — we don't track your usage, code, or conversations.
Cloud sync is opt-in , with delta tracking so you see exactly what's about to be pushed.

A pnpm monorepo with a shared core, plus submodules for the IDE, web platform, and companion app:
packages/
├── core/ # @ava/core — shared agent engine (agent loop, providers, tools, personas, memory)
├── cli/ # @ava/cli — terminal REPL
├── extension/ # VS Code extension host + React webview + dashboard
├── ide/ # Standalone IDE (Tauri v2 + Rust + Node.js engine) — submodule
├── mobile/ # Companion app — submodule
└── web/ # Platform website — submodule
Key design decisions:
Shared core, zero duplication — CLI, extension, and IDE all run the same @ava/core engine, tools, and providers.
Event-driven agent — the agent emits typed events every host subscribes to.
Self-contained tools — each tool carries its own schema and handler; adding one is a single file plus one registry line.
No provider SDKs — all provider communication uses native fetch .
Permission matrix — risk levels ( safe , write , dangerous ) combine with your permission mode to decide what needs approval. Plans and user questions always require approval.
Local-first — everything works offline; cloud sync is opt-in.
