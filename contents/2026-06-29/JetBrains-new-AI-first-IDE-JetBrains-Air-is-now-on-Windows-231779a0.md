JetBrains は、エージェント開発環境 (ADE) である JetBrains Air がダウで利用可能になったことを発表しました。

Windows x64 および ARM 上の nload。
JetBrains Air についてはあまり馴染みがないかもしれません。これは、開発者がすべての行を手動で記述する代わりに AI エージェントにタスクを引き渡せるようにするために、同社が 2026 年 3 月にリリースした新しいデスクトップ アプリです。これは、従来の統合開発環境 (IDE) ではなく、エージェント開発環境 (ADE) として見ることができます。後者は中央のテキスト エディターを中心に機能を構築しますが、前者は AI エージェント自体を中心にすべてを配置します。 JetBrains では次のように説明されています。
Air デスクトップ アプリは、ターミナルでのエージェントの実行と完全な IDE での作業の間のギャップを埋めます。エージェント開発専用の環境が提供され、タスクを実行し、結果を確認し、メインのコーディング ワークフローに集中し続けることができます。
Air は、VS Code と直接競争するのは負け戦だと悟った同社が 2025 年 12 月に密かに殺害した実験的エディター、Fleet の遺灰から誕生しました。同社は、Fleet の軽量で最新のアーキテクチャを再利用して Air を構築し、基本的なコード エディターを複数の AI エージェントを実行するためのワークスペースに変換しました。 Air がリリースされたとき、Air は macOS でのみ利用可能でした。 Linux ユーザーがこのソフトウェアを試す機会を得たのは、今月初めになってからでした。
Air が Windows 上で動作するようになったことで、コードを記述する前に計画モードで複雑な機能を計画したり、AI が実装計画をマークダウン ファイルに書き込むのを観察したりすることができます。この計画を繰り返し、特定のクラスまたはファイルへの参照を追加し、エージェントをローカルで実行するか、分離された Git ワークツリー内で実行するかを選択できます。エージェントを並行して実行すると、Codex が別のブランチでテストを作成している間に、Claude が 1 つのブランチでデータベース スキーマをリファクタリングできるため、他の重要な作業を自由に行うことができます。をセットアップすることもできます

クロードがコードを記述し、Codex がそれをレビューするパイプライン。
現時点では、Air は無料です。 JetBrains AI Pro または Ultimate サブスクリプションをお持ちの場合は、組み込みエージェントに完全にアクセスできますが、Anthropic、OpenAI、または Google に直接接続するために独自のキーを使用する (BYOK) オプションもあります。

JetBrains has announced that JetBrains Air, its Agentic Development Environment (ADE) is now available for download on Windows x64 and ARM.
You might not be familiar with JetBrains Air. It's this new desktop app that the company launched back in March 2026 to let developers hand off tasks to AI agents instead of writing every line manually. You can see it as more of an Agentic Development Environment (ADE) rather than a traditional Integrated Development Environment (IDE). The latter builds its features around a central text editor, while the former arranges everything around the AI agent itself. Here's how JetBrains describes it :
The Air desktop app fills the gap between running agents in the terminal and working in a full IDE. It gives you a dedicated environment for agentic development, where you can run tasks, review results, and keep your main coding workflow focused.
Air was born from the ashes of Fleet, an experimental editor that the company quietly killed in December 2025 after realizing that competing directly with VS Code was a losing battle. The company repurposed Fleet's lightweight, modern architecture to build Air, transforming a basic code editor into a workspace for running multiple AI agents. When Air launched, it was only available for macOS. It wasn't until earlier this month that Linux users got a chance to play with the software.
Now that Air is on Windows, you can do things like map out a complex feature in Plan mode and watch an AI write the implementation plan to a markdown file before writing any code. You can iterate on this plan, add references to specific classes or files, and choose whether to run the agent locally or inside an isolated Git worktree. Running agents in parallel means you can have Claude refactor a database schema in one branch while Codex writes tests in another, leaving you free to do other important things. You can even set up a pipeline where Claude writes the code, and Codex reviews it.
At the moment, Air is free. If you have a JetBrains AI Pro or Ultimate subscription, you get full access to the built-in agents, though there is also the option to bring your own key (BYOK) to connect directly to Anthropic, OpenAI, or Google.
