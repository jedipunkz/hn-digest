スキル
何をするのか
/portaljs-architect
アドバイザリー — 構築する前に、ニーズ (データ、規模、ガバナンス) を推奨アーキテクチャに変換します。スタックがわからない場合は、ここから始めてください。
/portaljs-新しいポータル
ブリーフから新しいポータル (ホーム + カタログ + ショーケース) をスキャフォールディングします。テンプレートをコピーし、プロジェクト名と説明を置き換え、deps をインストールし、ビルドを検証します。
/portaljs-add-dataset
CSV、TSV、JSON、または GeoJSON データセットを追加します。データセットをカタログに登録し、そのショーケースを自動的にレンダリングします。大きなローカルファイルは、Git LFS 経由で Cloudflare R2 にプッシュされます。
/portaljs-追加リソース
別のファイル (データ ディクショナリ、方法論、追加データ) を既存のデータセットに添付します。これはマルチリソースとなり、ショーケースはファイルごとにセクションをレンダリングします。
/portaljs-add-chart
折れ線グラフ、棒グラフ、面グラフ、円グラフ、または散布図をデータセットのショーケースに追加します。
/portaljs-add-map
GeoJSON データセットを対話型マップ上にレンダリングし、ホームページに登録します。
/portaljs-add-geo
自分のマシン上で地理空間ファイル (GeoJSON、Shapefile、GeoPackage、KML/KMZ、FlatGeobuf、ジオメトリ付き CSV) を自動取り込みます。サーバーなし: CRS を EPSG:4326 に正規化し、PMTiles レンダー層と GeoParquet クエリ層を導出し、両方とオリジナルを R2 にプッシュし、ショーケースのマップとクエリを適切な場所に配置する 1 つのデュアル層データセットを登録します。
/portaljs-define-schema
データセットのデータから摩擦のないテーブル スキーマを推測し、ライセンス/ソース/キーワードのメタデータを追加して、ショーケースに型付きフィールド テーブルを表示します。
/portaljs-add-dcat
ポータルを収集可能にする — 標準に準拠した DCAT フィード (DCAT 2) を発行します

/3、DCAT-AP、DCAT-US、国家プロファイル）を JSON-LD、Turtle、RDF/XML で作成できるため、国家/EU/米国のオープンデータ ポータルがそのデータセットを収集できます。
/portaljs-connect-ckan
静的ファイルではなく、API を介してポータルを CKAN バックエンドに接続します。
/portaljs-check-data-quality
データセットをそのスキーマに対して検証し、品質の問題 (型の不一致、欠損値、制約違反) にフラグを立てます。
/portaljs-移行
カタログ全体を収集または移行します。

Building a data portal has always meant more than a website. You have to decide where
the data lives, how it's versioned, how people search it, how it's served, and how it's
governed — and then wire a frontend on top. Teams either over-build on a heavy data
warehouse they don't need, or under-build on a pile of scripts that doesn't scale.
PortalJS is an open-source, agentic skills framework that helps data teams build,
develop, and ship data portals — and the data infrastructure underneath them. It isn't
only a frontend. The skills do two jobs:
Advise — given what you're building , what your data is , and what it's for ,
they recommend an architecture: storage, compute, catalog, access, hosting, metadata.
Build — they scaffold that stack as plain, editable Next.js code with no lock-in.
It is opinionated but open : the recommended modern path is git + object storage
(Cloudflare R2) + Parquet, queried with DuckDB — an open lakehouse
instead of a classic warehouse. For living, incremental tables you can layer on
DuckLake , and a traditional datastore (CKAN, a warehouse)
stays a first-class option when you need it. You always own plain code.

Three surfaces. Every data portal is built from three: a Home page that explains
it and offers search, a Catalog ( /search ) to discover datasets, and a Showcase
( /@<namespace>/<slug> ) to explore one dataset — metadata, preview, download/API, and
charts/maps. ( Core concepts → )
One seam. The surfaces read data only through a DataProvider , so the source — static
files today, a CKAN or lakehouse backend tomorrow — can change without touching a page.

PortalJS ships Claude Code skills that turn a brief into

Install the skills once into your personal scope so they're available from any
directory:
curl -fsSL https://raw.githubusercontent.com/datopian/portaljs/main/scripts/install-portaljs-skills.sh | bash
Restart Claude Code (or open a new session) and type / to see them. See
.claude/INSTALL.md for other install options (versioned plugin, or
running straight from a clone of this repo).
If you're not sure how to set up your portal, start with the advisor, then build:
/portaljs-architect we have ~200 public CSVs, updated quarterly, and must publish DCAT-AP
/portaljs-new-portal "Auckland Council open data portal"
/portaljs-add-dataset ./data/air-quality.csv
/portaljs-add-dataset https://example.com/parks.geojson
The skills are interactive — if your brief is thin, they interview you in short rounds
rather than erroring. /portaljs-architect recommends a stack and hands off; /portaljs-new-portal
scaffolds the three surfaces; /portaljs-add-dataset appends to the datasets.json manifest and
the showcase renders automatically at /@<namespace>/<slug> . Run npm run dev and you
have a portal.
Prefer to build by hand? The skills are a convenience, not a requirement — scaffold the
template directly with the CLI:
npm create portaljs@latest my-portal
(Or grab the bare template with no prompts: npx tiged datopian/portaljs/examples/portaljs-catalog my-portal .)
Skill
What it does
/portaljs-architect
Advisory — turns your needs (data, scale, governance) into a recommended architecture before you build. Start here if you're unsure of the stack.
/portaljs-new-portal
Scaffold a new portal (Home + Catalog + Showcase) from a brief — copies the template, substitutes your project name and description, installs deps, verifies the build.
/portaljs-add-dataset
Add a CSV, TSV, JSON, or GeoJSON dataset — registers it in the catalog and renders its showcase automatically; large local files are pushed to Cloudflare R2 via Git LFS for you.
/portaljs-add-resource
Attach another file (data dictionary, methodology, extra data) to an existing dataset — it becomes multi-resource and the showcase renders a section per file.
/portaljs-add-chart
Add a line, bar, area, pie, or scatter chart to a dataset's showcase.
/portaljs-add-map
Render a GeoJSON dataset on an interactive map and register it on the home page.
/portaljs-add-geo
Auto-ingest a geospatial file (GeoJSON, Shapefile, GeoPackage, KML/KMZ, FlatGeobuf, CSV-with-geometry) on your own machine — no server: normalizes CRS to EPSG:4326, derives a PMTiles render tier and a GeoParquet query tier, pushes both plus the original to R2, and registers one dual-tier dataset the showcase maps and queries in place.
/portaljs-define-schema
Infer a Frictionless Table Schema from a dataset's data, add license/source/keyword metadata, and surface a typed field table on its showcase.
/portaljs-add-dcat
Make the portal harvestable — emit standards-compliant DCAT feeds (DCAT 2/3, DCAT-AP, DCAT-US, national profiles) in JSON-LD, Turtle, and RDF/XML so national/EU/US open-data portals can harvest its datasets.
/portaljs-connect-ckan
Wire the portal to a CKAN backend over its API instead of static files.
/portaljs-check-data-quality
Validate a dataset against its schema and flag quality issues (type mismatches, missing values, constraint violations).
