No developer tools needed. Install from the Chrome Web Store, then connect
a model — a provider with a free tier, your own key, or a free local app like Ollama.
The last step (the helper app) is optional and only needed for local Claude Code, Codex & Antigravity.
Download
chatpanel-extension.zip
and unzip it to a folder you’ll keep.
Load it. Open chrome://extensions (Edge:
edge://extensions ), turn on Developer mode ,
click Load unpacked , and pick the unzipped folder
(the one with manifest.json ).
Connect a model. Open Settings → API, choose a provider,
add your sign-in or key, load the model list, pick one, and click Test.
(Optional) Install the Bridge helper app — it lets the
browser extension drive your local Claude Code, Codex &
Antigravity . Runs quietly in the background; ChatPanel finds it automatically.
# macOS / Linux — one line, no extra tools needed
curl -fsSL https://dl.chatpanel.net/bridge/install.sh | bash
# Windows (PowerShell) — no Node needed
irm https://dl.chatpanel.net/bridge/install.ps1 | iex
Just want to try it once? npx @chatpanel/bridge runs it in the foreground.
The --install step keeps it running at login.
(Optional, advanced) Install the Privacy Gateway — a local, privacy-focused
model router . It redacts every request with a bundled NER
(names, orgs, locations) and routes the cleaned request to either a model API
(OpenAI/Anthropic-compatible) or your local CLI agents via the Bridge. Any
OpenAI-compatible client — a terminal agent (OpenCode, Pi, aider) or the ChatPanel
extension itself — can point at it.
# macOS / Linux
curl -fsSL https://dl.chatpanel.net/gateway/install.sh | bash
# Windows (PowerShell)
irm https://dl.chatpanel.net/gateway/install.ps1 | iex
# or via npm (any OS, needs Node)
npm i -g @chatpanel/gateway && chatpanel-gateway --install # redaction proxy on http://127.0.0.1:4320
Point any OpenAI-compatible client at http://127.0.0.1:4320/v1 ; the gateway redacts
on the way out and restores on the way back, then forwards to a model API (your
key) or your subscription agents via the Bridge. More in the FAQ.
Bridge vs Gateway? The Bridge is lightweight — its one job is
connecting the browser extension to your local CLI agents; most people only need this.
The Privacy Gateway is a separate, privacy-focused model router : it
redacts with a bundled NER and routes the cleaned request to a model API or your CLI
agents — and any OpenAI-compatible client, including the extension, can point at it. Three
independently-versioned pieces (extension, bridge,
gateway); install only what you use.
