## Original Extract

foomchat is an AI chat that builds installable UI features on request — describe a tool, install it live, and submit your best to the curated marketplace.

I had this idea a while ago: these language models write decent code now — enough to quickly prototype different ways their own interfaces could work.
So why keep reinventing the wheel separately? Combine it: a chat interface where, within the interface, you simply ask the model for the interface changes you want. That's Foomchat. You ask for messages rendered a certain way, a font, your own theme, a timer in the sidebar — you describe it, the model codes it, and it's installed live in the app. Maybe it isn't perfect the first run — models still misinterpret things — so you explain the change you want, it updates the feature, done. To publish a feature, it first has to be reviewed — features are, in essence, arbitrary code injection, so it's vital that everything on the browse page is safe. Once approved, a feature goes public on the catalog. Cards have a "try" button that installs it locally without saving it to your account (refresh and it's gone), and an install button that makes it permanent — from then on, whatever it modifies is just part of the site for you. Published features have dependency tracking and version locking, so features can extend other features without a dependency update breaking yours. New signups get 100 free credits, enough for a real taste. You can bring your own API key, or there are $10 and $20 subscriptions for the convenient route. Solo project. I'm looking for people to give it a try and hopefully find real use in it — and if there's anything to critique, I genuinely want to hear it so I can make it better.
