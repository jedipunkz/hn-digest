私は 22 年以上にわたり、MicroVAX II で実行できる 1980 年代の MATLAB バージョンを探してきました。
私が発掘できたのは、コンパイルされて問題なく動作した、1982 年の最初のリリース (FORTRAN ソース ファイルのコレクション) だけでした。
しかし、私が探していたバージョンはそれより後のバージョン、MATLAB 3.x あたりでした。これは GUI 以前の、ユーザー関数、スクリプト ファイルの実行、信号処理ツールボックスなどの多くの機能を導入したコマンド ラインのみの製品でした。
私はあらゆるところを調べて尋ね、さらにはクリーブ・モラー自身に数回連絡を取りました。

数年前。彼は検索と古いコードを自分で実行することに興味を持っていましたが、最初の連絡以降連絡はなく、残念ながら数か月前に亡くなりました。
2018 年のこの投稿で、彼は「PC-MATLAB の最初のバージョンのディスクが見つかりません。私が見つけることができる最も古いものはバージョン 1.3 です。そのバージョンのすべての関数、キーワード、および演算子は次のとおりです。」と述べています。
時が経ち、このプロジェクトは私の頭の中にありましたが、完了するにはあまりにも多くの時間と労力がかかるため、このプロジェクトはまったく非現実的でした。しかし、そこにクロードがやって来ました。
私はその機能のリストを手元に持っていました、そしてクロード。そこで私は、その仕様に合わせて MATLAB の大胆な「クリーンルーム再実装」を試みる機が熟したと判断しました。ただし、言語を簡単に拡張できる M ファイル実行機能が追加されました。独自の関数を作成することで、独自の新しいツールボックスを作成し、言語を無期限に拡張できます。バージョン 3.5 に十分近いため、同様に使用できます。
私は午後の数時間を Claude Code と過ごし、作業方法について最小限の指示を与えました。ただ、私が持っているコンパイラと機能のリストを (彼に?) 伝え、途中で出てくる質問に答えるだけでした。最終的には、その機能リストに加えて、私が要求した (なぜそうしないのか) REGIS/SIXEL グラフィックス出力を実装する完全なクローンが完成しました。クロードは、環境のスモークテストから最初から最後まで素晴らしい仕事をしてくれました。

- 最終データ: argv[1] は、RUN と外部コマンドの呼び出しの両方で大文字と小文字を保持します。 LIB$GET_FOREIGN は常に大文字になります。これで設計上の決定は決まりました。大文字と小文字を区別するファイル名には LIB$GET_FOREIGN ではなく argv を使用してください。
